취약점 분석

저는 트랙 선택 시 차후 제가 현업에서 하고 싶은 업무를 고려했습니다! 하드웨어부터 SW까지 광범위한 영역에서의 취약점을 분석하고 보안 조치를 취하는 업무를 하고 싶기 때문에 취약점 분석 트랙을 선택했습니다. BoB는 7~8월 간의 교육기간이 있는데, 공통교육과 트랙교육으로 구성되어 있습니다. 트랙 교육 시에 선택한 트랙과 관련된 수업을 듣게 되는데, 취약점 분석 트랙 수업을 들으며 기술 수준을 높이고자 선택했고, 현재 여러 수업을 들어본 결과 취약점 분석 트랙 수업이 기술적으로 상당히 깊고 넓게 진행되기 때문에 아주 만족스럽게 생각하고 있습니다 :D

제가 생각할 때 1학년부터 4학년까지 개인의 수준에 따라 다음 단계들을 적절히 분배해서 진행하는 게 좋을 것 같습니다.
[1단계] CS 기본 및 개발 지식 습득 → [2단계] 각종 개발 및 보안 팀프로젝트 진행 → [3단계] 인턴 및 대외활동 혹은 대학원 랩실 학부연구생 단계를 밟으면 좋을 것 같습니다.

[1단계] - 1학년

우선 1학년 때는 간단한 토이 프로젝트를 해보며 CS 기본 지식과 개발 지식을 쌓는 것이 좋을 것 같습니다. python과 C(혹은 C++)로 간단한 프로그램 개발(ex. 간단한 웹페이지 제작, 간단한 계산기 프로그램 구현, 간단한 실행파일 제작)을 병행하면서 궁금한 지식들이나 기반 지식들을 쌓는 것이 좋을 것 같습니다. 개발을 직접 해보는 경험이 있으면 수업시간에 배우는 지식들을 이해하는 데에도 도움이 되는 것 같습니다. 그리고 가능하시다면, 1학년 때 알고리즘 공부를 해보는 것도 좋은 것 같습니다!


[2단계] - 2학년

2학년 때는 본격적으로 공모전, 대회 등을 참여하며 팀프로젝트를 진행하는 것이 좋을 것 같습니다. 1학년 때 어느 정도 지식을 쌓았다면 관심 분야에 대한 본격적인 프로젝트를 진행해볼 수 있을 겁니다! 이때 관심 가는 것, 재밌어 보이는 것들을 최대한 많이 동기들과 함께 진행하는 것을 추천드립니다. 2학년 때는 부담 없이 하고 싶은 것을 마음껏 해보기에 가장 좋은 시기라고 생각합니다! 대회를 꼽아보자면 Girls in ICT, 소프트웨어 개발 보안 경진대회를 추천드리지만, 사실 어떤 대회든 도움이 많이 되는 것 같습니다!


[3단계] - 3,4학년

3학년, 4학년 때는 인턴 혹은 대외 활동을 추천드립니다! 2학년때 어느 정도 개발이나 프로젝트 경험을 쌓고 나면, 3학년과 4학년때 인턴이나 대외활동에서 선발될 수 있는 스펙이 쌓였을 것입니다. 인턴과 대외활동을 엮어서 추천드리는 이유는 결국 두 활동 다 현업과 가까운 수준의 프로젝트를 진행할 수 있기 때문입니다.

[추천하는]

구체적인 추천은 2가지 드릴 수 있을 것 같습니다.

(1) ICT 학점연계인턴쉽 - 인턴쉽 진행 기관이 대부분 중소기업 혹은 중견기업입니다. 큰 기업들에 비해 작은 기업들에서는 인턴에게도 본격적인 업무를 부여하는 경우가 많은 것 같습니다. 물론 조금 고생스럽기도 했지만, 인턴이라는 직책에 비해 중요도가 높은 업무를 하는 과정에서 많이 성장했고, 사무국에서 여러 장치(ex.계약, 설문조사, 기업 내 담당자와 소통 등)로 늘 기업들을 감시하고 있기 때문에 추천드립니다!

(2) BoB - 아직 수료는 하지 않았고 교육을 받고 있지만, 교육만으로도 저는 BoB 활동을 추천드립니다. 교육이 정말... 재밌습니다!! 다양한 분야를 빠르게 학습할 수 있어서 빠르게 지식을 쌓을 수 있는 것 같습니다!!
